# ZenPulse: AI Meditation App

Мобильное приложение для медитации с AI-генерацией персонализированных аффирмаций.

## Запуск

```bash
npm install
npx expo start
```

Отсканируйте QR-код в Expo Go или нажмите `w` для запуска в браузере.

## Стек

- React Native + Expo (SDK 55)
- Expo Router (файловая навигация)
- TypeScript
- expo-linear-gradient, expo-haptics, AsyncStorage

## Экраны

### 1. Meditations (Главная)
Список медитаций в виде карточек с картинками и длительностью. Карточки без подписки отображаются с замком и серым оверлеем. При нажатии на заблокированную карточку — переход на Paywall. Фильтрация по категориям (All / Focus / Sleep / Calm / Energy).

### 2. Paywall (Подписка)
Модальный экран с градиентным фоном. Список преимуществ Premium, выбор тарифа (Monthly $9.99 / Annual $59.99 с бейджем "Save 50%"). Кнопка "Try 7 Days Free" имитирует покупку: loading state → сохранение в AsyncStorage → alert → возврат на главную с разблокированным контентом.

### 3. AI Mood Check-in
Выбор настроения из трёх эмодзи (😌 Calm / 😰 Anxious / 😔 Low). После выбора — имитация AI-генерации (1.5-2.5 сек) и показ персонализированной аффирмации с анимацией fade-in. Кнопка "Generate Another" для повторной генерации.

## Архитектура

Подписка управляется через React Context (`SubscriptionContext`). Флаг `isSubscribed` хранится в AsyncStorage и восстанавливается при перезапуске. Все экраны получают доступ к статусу подписки через хук `useSubscription()`.

Paywall зарегистрирован как модальный экран в root layout (`presentation: 'modal'`) — всплывает поверх табов снизу вверх.

## Как AI справился с мобильной спецификой

- **Навигация:** Expo Router с файловой структурой — AI корректно создал layout-файлы и модальную маршрутизацию
- **SafeArea:** Использованы разные edges для разных контекстов — `['top']` для экранов с табами (таб-бар сам обрабатывает нижний safe area), `['top', 'bottom']` для модального paywall
- **Haptics:** Тактильный отклик на кнопках — light impact на выбор эмодзи, medium на покупку подписки
- **Адаптивность:** Карточки медитаций используют пропорциональную высоту (`width * 0.45`), чипы категорий в горизонтальном ScrollView, flex-layout вместо фиксированных размеров

## Контрольный вопрос

**С какими специфическими проблемами мобильной вёрстки AI справляется хуже всего и как контролировалась его работа?**

1. **SafeArea** — AI склонен ставить `edges={['top', 'bottom']}` на всех экранах, хотя для экранов внутри TabNavigator нижний edge не нужен (таб-бар сам учитывает home indicator). Требуется ручная проверка каждого экрана.

2. **Touch targets** — AI генерирует кнопки по размеру текста (например, 30×20pt), что непригодно для пальцев. Нужно вручную добавлять `minHeight: 44`, `hitSlop` и достаточные паддинги.

3. **Фиксированные размеры** — AI использует `position: absolute` с пиксельными значениями вместо `flex`. На iPhone SE (375×667) элементы налезают друг на друга, на Pro Max (430×932) — остаются слишком мелкими. Решение: flex-layout, процентные значения и `Dimensions.get('window')`.

4. **Home indicator** — AI путает `paddingBottom` с учётом и без учёта safe area. На iPhone без кнопки Home это приводит к тому, что нижние элементы перекрываются.

**Контроль:** проверка каждого экрана в Expo Go на реальном устройстве после написания, сверка с разными размерами экранов через эмулятор.
